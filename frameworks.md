# FRAMEWORKS

## F01: Worldbuilding Framework

**Purpose:** Construct a shared reality space that makes your message inevitable rather than arguable.

### Inputs
- Target audience identity, beliefs, language, desires
- Your message/proposition
- Available examples, case studies, prototypes

### Process

```
STEP 1: MAP THE EXISTING WORLD
  → What does the audience currently believe?
  → What language do they use?
  → What values do they hold?
  → What identity do they protect?

STEP 2: FIND ATOMIC UNITS
  → Identify non-controversial foundations they already accept
  → These are the "nod" points

STEP 3: CHOOSE ENTRY DIRECTION
  → ZOOM IN: Universe → Continent → City → Location (for broad audiences)
  → ZOOM OUT: Location → City → Continent → Universe (for expert audiences)

STEP 4: ESTABLISH SHARED GROUND
  → Present atomic units sequentially
  → Each "nod" builds foundation for the next claim

STEP 5: INTRODUCE MODIFICATION
  → After sufficient agreement, introduce your deviation
  → Frame it as a natural extension, not a contradiction

STEP 6: ROOT WITH EXAMPLES
  → Anchor modification with concrete instances
  → More examples = stronger rooting
  → Prototype if no examples exist

STEP 7: ADD MICRO-DETAILS
  → Consistent small details reinforce internal logic
  → The deeper you go, the more believable the world

STEP 8: MAINTAIN CONSISTENCY
  → Every element must obey established rules
  → One inconsistency can collapse the entire world
```

### Failure Modes
- Starting with your modification before shared ground → rejection
- Wrong audience language → outsider signal
- No examples → unrooted, theoretical
- Inconsistencies → immersion break
- Too many modifications at once → cognitive overload

## F02: Frame Shift Framework

**Purpose:** Change an existing belief without direct confrontation.

### Process

```
STEP 1: ENTER THEIR WORLD COMPLETELY
  → Use their language, validate their beliefs, demonstrate membership

STEP 2: GENERATE AGREEMENT CASCADE
  → Present claims they already agree with (nod sequence)
  → Each nod lowers defenses further

STEP 3: INTRODUCE THE PIVOT
  → "By the way..." / "But consider..." / "What if..."
  → Present modification as natural extension

STEP 4: ROOT THE MODIFICATION
  → Immediately provide concrete examples
  → Stack examples — the more, the stronger

STEP 5: LET THEM RESOLVE
  → If executed correctly, audience resolves the shift internally
  → They feel they discovered the idea, not that you sold it
```

### Failure Modes
- Entering wrong world (misidentified audience)
- Pivoting too early (insufficient agreement cascade)
- Zero examples after pivot (unrooted modification)
- Combative framing ("you're wrong about X")

## F03: Audience Deconstruction Framework

**Purpose:** Reverse-engineer the target to calibrate every element of your message.

### Process

```
STEP 1: MAP IDENTITY
  → What do they believe about themselves?
  → What self-image do they protect?

STEP 2: MAP PSYCHOLOGY
  → What are their deepest hopes? Fears?
  → What do they optimize for? (conviction, profit, safety, status)

STEP 3: MAP LANGUAGE
  → What in-group terms define their tribe?
  → What jargon signals membership?

STEP 4: MAP WORLDVIEW
  → What do they believe about how the world works?
  → What are their atomic units?

STEP 5: MAP EXAMPLES
  → What precedents and heroes do they accept?
  → What case studies have shaped their beliefs?

STEP 6: DESIGN MESSAGE AS MIRROR
  → Reflect their identity, speak their language, fit their world
  → Address psychological needs, not just technical needs
```

## F04: Leverage Point Framework

**Purpose:** Win arguments with minimum force by identifying the single root assumption.

### Process

```
STEP 1: MAP SURFACE ARGUMENTS
  → List all visible reasons, symptoms, objections

STEP 2: TRACE DEPENDENCIES
  → For each: "What must be true for this to hold?"

STEP 3: FIND THE ROOT
  → The single assumption that, if changed, collapses everything

STEP 4: ATTACK THE ROOT
  → Present evidence or reframe against ONLY that assumption

STEP 5: IGNORE THE REST
  → Surface symptoms dissolve when the root shifts
```

### Failure Modes
- Mistaking a surface symptom for the root
- Attacking multiple points (force dilution)
- Engineering answer syndrome (listing many reasons)

## F05: Dissonance Resolution Framework

**Purpose:** Create a cognitive opening by surfacing a contradiction in the target's own beliefs.

### Process

```
STEP 1: ESTABLISH BELIEF A
  → "You believe X is true, right?" (get confirmation)

STEP 2: ESTABLISH BELIEF B
  → "You also believe Y is true, right?" (get confirmation)

STEP 3: REVEAL CONTRADICTION
  → Show how X and Y cannot both be true

STEP 4: CREATE TENSION
  → Let the dissonance sit — the audience NEEDS to resolve it

STEP 5: OFFER RESOLUTION
  → Present your idea as the solution to the tension
  → The audience adopts it as relief, not imposition
```

## F06: AI Universe Construction Framework

**Purpose:** Produce high-quality AI outputs by treating prompts as worldbuilding.

### Process

```
STEP 1: SET PHYSICAL LAWS
  → Define constraints, rules, boundaries (system prompt)
  → Declare what the AI must NOT do (anti-patterns)

STEP 2: DEFINE THE UNIVERSE
  → Specific entities, relationships, context
  → The more dimensions specified, the richer the output

STEP 3: PROVIDE EXAMPLES
  → Few-shot examples establish patterns for inference
  → Examples = rooting for AI

STEP 4: ADD MICRO-SPECIFICS
  → The more specific your world, the richer the output
  → Every unspecified dimension defaults to statistical average

STEP 5: ITERATE AND DEEPEN
  → Refine the world with each interaction
  → Add laws as you discover unwanted default behaviors
```

### Failure Modes
- Three-line prompt → generic universe → generic output
- Shaping only one dimension → rest defaults to generic
- No examples → AI infers from broadest distribution
- No anti-patterns → AI produces its default tendencies
