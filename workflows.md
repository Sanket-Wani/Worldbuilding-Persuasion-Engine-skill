# WORKFLOWS

## W01: The Complete Persuasion Workflow

Use this end-to-end workflow whenever you need to change someone's mind, sell an idea, write a pitch, or construct a convincing narrative.

```
PHASE 1: RECONNAISSANCE
  1. Identify the target (person, group, AI model)
  2. Run Audience Deconstruction Framework (F03)
     → Map: identity, psychology, language, worldview, accepted examples
  3. Identify the world they inhabit
  4. Document atomic units (beliefs they won't argue with)

PHASE 2: WORLD ENTRY
  5. Choose zoom direction (DT02)
     → Broad audience → Zoom In
     → Expert audience → Zoom Out
  6. Open with atomic units — generate the nod sequence
  7. Use their language, their frames, their examples
  8. Establish yourself as a member of their world

PHASE 3: MODIFICATION
  9. Choose persuasion method (DT04)
     → Frame Shift if modifying incrementally
     → Dissonance if surfacing contradictions
     → Leverage Point if attacking a root assumption
  10. Introduce your deviation as natural extension
  11. Root IMMEDIATELY with concrete examples
  12. Stack examples for density

PHASE 4: DELIVERY
  13. Choose complexity level (DT05)
  14. Choose argument count (DT06) — prefer ONE devastating over MANY mediocre
  15. Wrap data in story frames (DT08)
  16. Ensure cognitive hospitality throughout
  17. Check consistency of your world at micro level

PHASE 5: VERIFICATION
  18. Does every element serve the receiver's experience?
  19. Is the cognitive immune system managed throughout?
  20. Is the world rooted with sufficient examples?
  21. Is there one clear resolution / call to action?
```

## W02: The Cold Outreach Workflow

Use this for emails, DMs, pitches, applications — any unsolicited communication.

```
1. Research the target thoroughly
   → Company name, their language, their challenges, their identity

2. Open with proof of homework
   → Mention specific details only an insider would know

3. Signal tribal membership
   → Use in-group language
   → Reference shared values/culture

4. Present yourself as their mirror
   → Show you embody what they already value
   → "Strategic self-deprecation" — frame weaknesses as the traits they fetishize

5. Address psychological needs, not technical ones
   → They don't want features; they want confirmation of their worldview

6. Keep it short — cognitive hospitality
   → Every unnecessary word raises cognitive load
```

## W03: The AI Prompting Workflow

Use this every time you prompt an AI model.

```
1. DEFINE THE UNIVERSE
   → What world should this AI inhabit for this task?
   → What are the laws of this universe?

2. SET PHYSICAL LAWS
   → Constraints, boundaries, forbidden behaviors
   → Style rules, tone rules, format rules

3. POPULATE THE WORLD
   → Entities, relationships, context, backstory
   → The more dimensions specified, the fewer generic defaults

4. PROVIDE EXISTENCE PROOFS
   → Few-shot examples of desired output
   → These ARE your rooting — they teach the AI your world's rules

5. DECLARE ANTI-PATTERNS
   → Explicitly state what the AI must NOT do
   → "Do not use negation clichés"
   → "Do not open with questions"

6. ADD MICRO-SPECIFICS
   → The granularity of your world determines the richness of output
   → Every unspecified dimension = generic default

7. ITERATE
   → If output drifts from your world, add more laws
   → If output is too constrained, relax some laws
   → Each iteration deepens the universe
```

## W04: The Pitch Construction Workflow

Use this for investor pitches, business proposals, project pitches.

```
1. Map the investor/decision-maker's world
   → What mathematics do they operate on?
   → What outcomes do they need?
   → What language defines their tribe?

2. Start in THEIR world with THEIR language
   → Use their jargon (GTM, ARR, CAC, etc.)
   → Validate their existing beliefs

3. Build the nod sequence
   → Present problems they already agree exist
   → Each nod lowers defenses

4. Introduce your frame shift
   → "What if there's an approach that solves this differently?"
   → Frame as natural extension, not contradiction

5. Root with examples
   → Show prior art, case studies, comparable successes
   → If no examples exist, show a prototype
   → If no prototype exists, acknowledge the first-mover cost

6. Concentrate on ONE leverage argument
   → Not five reasons — ONE devastating reason
   → The one that maps to their deepest optimization target

7. Close with identity confirmation
   → "You're the kind of investor/leader who sees this"
   → Mirror their desired self-image
```
