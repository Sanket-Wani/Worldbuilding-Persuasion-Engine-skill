# MENTAL MODELS

## MM01: Communication as Program Execution

Writing is not transmitting a message. It is executing a program in another entity's operating system.

- The communicator is a **programmer**
- The audience's mind is the **runtime environment**
- The message is **code**
- Success = the program runs correctly in THEIR environment
- Failure = asking them to download YOUR operating system

**Application:** Before writing anything, ask: "What operating system is my receiver running? Will my program execute cleanly on it, or will it crash?"

---

## MM02: The Cognitive Immune System

The brain operates an active defense system against unfamiliar information, analogous to biological immunity.

**Immune response formula:**
```
Response Strength = Novelty × Cognitive Load × Distance from Existing Beliefs
```

**Immune suppressors:**
- Prior validation of existing beliefs
- High-quality presentation (reduces perceived threat)
- Simple, familiar language
- Emotional vulnerability / lowered defenses

**Application:** Every piece of communication must manage immune response. If defenses are high, validate first. Never introduce novelty when defenses are up.

---

## MM03: Worlds as Operating Systems

Every person, community, culture, and AI model operates within a "world" — a self-contained reality with its own rules, language, values, and examples.

**Key properties:**
- Worlds have atomic units (foundational non-controversial beliefs)
- Worlds have in-group language (tribal signals)
- Worlds have heroes and precedents (accepted examples)
- You cannot build a world entirely from scratch — all worlds inherit from existing ones

**Application:** Before communicating, map the target world: What are its atomic units? What language does it use? What examples does it accept? What identity does it protect?

## MM04: Examples as Existence Proofs

An idea without a concrete example is fundamentally unreal. Examples don't merely illustrate — they make things EXIST.

**Hierarchy of rooting strength:**
1. Multiple real-world case studies (strongest)
2. Single real-world case study
3. Working prototype
4. Analogous example from another domain
5. No examples (unrootable — do not proceed)

**Application:** The strength of any claim = number × quality of rooting examples. If you have zero examples, either build a prototype or wait.

---

## MM05: The First-Mover Paradox

Being first means having no examples to root your story with, which means the persuasion economics are terrible.

**The cycle:**
```
No prior examples → Can't root the story → Can't raise support → Must self-fund →
Succeed → Become the example → Unlock the entire category for others
```

**Application:** If you're first, accept the persuasion cost. Build the proof yourself. Once you succeed, you become the case study everyone else points to.

## MM06: Prompts as Universe Spawning

Every AI prompt creates a temporary universe with its own physics.

**Technical mapping:**
| Human Communication | AI Prompting |
|---|---|
| Worldview | Embedding space region |
| Rules of the world | System prompt constraints |
| Examples/case studies | Few-shot examples |
| In-group language | Domain-specific vocabulary |
| Micro-details | Specificity tokens |
| Cognitive immune system | Statistical default distribution |

**Application:** A three-line prompt is a god creating a barren universe. A detailed prompt with laws, examples, entities, and anti-patterns is a god creating a rich, specific reality.
