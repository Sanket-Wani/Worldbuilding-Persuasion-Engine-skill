# TEMPLATES

## T01: Cold Email / Outreach Template

```
[PROOF OF HOMEWORK — mention their company/project by name, reference something specific]

[IDENTITY MIRROR — present yourself as embodying what they value most]

[STRATEGIC SELF-DEPRECATION — frame a "weakness" that is actually the trait they fetishize]

[IN-GROUP LANGUAGE — use their tribal vocabulary to signal membership]

[SINGLE ASK — one clear, low-friction next step]
```

### Structure Logic:
- Line 1 shifts conversation from your wants to their world
- Line 2 activates confirmation bias (you ARE who they want)
- Line 3 lowers defenses through vulnerability while signaling obsessive focus
- Line 4 signals tribal membership
- Line 5 makes action easy

## T02: Frame Shift Argument Template

```
[ENTER THEIR WORLD]
You're right that [validate their current belief]. Most [their group] agree because [reason they believe it].

[NOD SEQUENCE]
And it makes sense because [atomic unit 1]. And [atomic unit 2]. And we all know [atomic unit 3].

[PIVOT]
But here's something interesting: [introduce your modification as a natural extension].

[ROOT]
For example, [concrete example 1]. And [concrete example 2]. And [concrete example 3].

[RESOLUTION]
Which means [your conclusion follows naturally from their own logic + your modification].
```

## T03: Dissonance Argument Template

```
[ESTABLISH BELIEF A]
You believe [X], right? [Get confirmation or state something undeniable]

[ESTABLISH BELIEF B]
You also believe [Y], right? [Get confirmation or state something undeniable]

[REVEAL CONTRADICTION]
But if [X] is true, then [consequence that conflicts with Y].
And if [Y] is true, then [consequence that conflicts with X].
Both can't be true simultaneously.

[TENSION]
So something has to give.

[RESOLUTION]
Here's what resolves it: [your idea that reconciles both, or replaces one].
```

## T04: Leverage Point Argument Template

```
[ACKNOWLEDGE THE SURFACE]
Most people argue about [surface symptom 1] and [surface symptom 2] and [surface symptom 3].

[GO DEEPER]
But all of these depend on one assumption: [state the root assumption].

[CHALLENGE THE ROOT]
Let's examine that assumption: [evidence or reframe that undermines it].

[COLLAPSE]
If that assumption doesn't hold, then [surface symptom 1] doesn't matter,
[surface symptom 2] doesn't matter, and [surface symptom 3] doesn't matter.

[REBUILD]
Instead, [your alternative, built on a different root assumption].
```

## T05: Story-Wrapped Data Template

```
[CHARACTER]
[Name], [role/context], used to [old behavior with pain point].

[TRANSFORMATION]
Then [intervention/change happened].

[RESULT]
Now [specific measurable outcome]. That's [data point in relatable scale].

[IMPLICATION]
Which means [broader principle this illustrates].
```

### Example:
"Himish, one of our top reps, used to spend every Friday afternoon updating spreadsheets. Now that process is automated, and last Friday he closed two extra deals. That's five lakhs in additional revenue from one person in one afternoon."

